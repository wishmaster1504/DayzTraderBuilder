Входные параметры:
- Можно задать путь к конфиг файлу TraderBuilder.config без указанием его имени (с.да будем генерить файл трейдера).


Что надо:
Конфиг файл (по умолчанию используем имя TraderBuilder.config)
Конфиг файл с суммами md5 по трейд файлам (автогенерируемый с режимом перезаписи MD5 после обновления файлов)
Логирование в файл надо? Сделано

Ограничения:
- создать папку, в которую положить (можно в папке с трейдером):
   1. Файл запуска программы
   2. Создать папку "/OUT"
   3. В папку "/PathTraderFiles" положить файлы всех трейдеров (сюда же будем сохранять файл с MD5)
   4. Готовый файл трейдера будет сформирован в папку где лежит конфиг программы
   5. В именах файлов трейдера НЕ использовать пробелы
    
Содержание конфиг файла
- путь к файлам трейдера <PathTraderFiles>
- список файлов трейдера (задает порядок добавления их в основной файл) <FilesList><File>ИмяФайла.txt</File></FilesList>
- основное имя файла трейдера (TraderConfig.txt) <MainFileName
- полный путь куда копировать сгенерированный (ЕСТЬ возможность задать больше 1-го пути. Например для двух одинаковых серверов)
  <PathCopyTo><Path><Path></PathCopyTo>

Содержание Конфиг файл с суммами md5 по трейд файлам (сохраняется в папку с конфигом)
- автогенерируемые записи суммы MD5 по каждому файлу трейдера


----- Алгоритм -----

Определяем текущий каталог, откуда вызвана программа.
Читаем конфиг файл
Проверяем изменялись ли файлы трейдера и не добавились ли новые (MD5)
Если файлы не менялись, то завершаем работу (новый файл не генерим)

Удаляем существующий файл трейдера в папке /IN (если есть)
Создаем файл трейдера в папке /IN
Пишем начальный тег трейдера
Проходим по каждому элементу массива файлов
Читаем файл и тупо копируем все содержимое (в режиме добавления)
Пишем заключительный тег файла трейдера (EndFile)
Закрываем файл трейдера

Копируем созданный файл трейдера в папки, указанные в конфиг файле (может быть указано несколько путей)

Перезаписать новые суммы MD5 по файлам в конфиг MD5

Конец программы
